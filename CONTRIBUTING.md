# Contributing to Refactor Pilot Framework

Thanks for considering a contribution. This project benefits from community input — whether it's improving prompts, adding scripts for new languages, sharing examples, or fixing typos.

## How to Contribute

### Report Issues

If you found a problem with a prompt, script, or documentation:

1. Open a GitHub issue
2. Describe what you tried, what you expected, and what happened
3. Include the AI tool and model you were using (if relevant)

### Suggest Improvements

If a prompt could be better or a workflow step is missing:

1. Open a GitHub issue with the "enhancement" label
2. Describe the improvement and why it matters
3. If possible, include a before/after example

### Submit Changes

1. Fork the repository
2. Create a branch (`git checkout -b improve-phase2-prompts`)
3. Make your changes
4. Test your changes (run scripts, try prompts with an AI tool)
5. Commit with a clear message
6. Open a pull request

### What We're Looking For

- **Better prompts** — Prompts that produce more consistent, higher-quality AI output
- **New language support** — Scripts and prompts tailored for Python, Go, Rust, Java, etc.
- **Real-world examples** — Anonymized examples of the framework applied to actual projects
- **Claude skills improvements** — Better skill descriptions, additional workflow automation
- **Documentation** — Clearer explanations, better diagrams, more tips

### Guidelines

- Keep prompts tool-agnostic. They should work with Claude, ChatGPT, Copilot, Gemini, and any other AI tool.
- Scripts should be dependency-free where possible (Node.js standard library, Python standard library).
- Test your prompts with at least two different AI models before submitting.
- Keep the tone practical and direct. No fluff.

## Code of Conduct

Be respectful. Be constructive. We're all here to make refactoring less painful.
